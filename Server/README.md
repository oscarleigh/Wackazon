# Wackazon Backend

## Installing Go
- Make sure you have [Homebrew](https://brew.sh) installed.
- Note to Oscar: you already have it installed.
- Once you have Homebrew installed, install Go by entering into your terminal:
```bash
brew install go
```

## Running the server
- Ensure you are in the Server directory on the terminal.
- Type the command below to run the backend server:
```bash
go run main.go 
```

## Accessing the server
- The backend server is accessible on `http://localhost:8080`.
- It is only accessible on your machine, so to use it on the Wackazon SwiftUI frontend,
you have to use something like [ngrok](https://ngrok.com/) or [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/networks/connectors/cloudflare-tunnel/get-started/create-remote-tunnel/) to tunnel to your machine.
- If using ngrok, then once the backend is running, run ngrok by:
```bash
ngrok http 8080
```
- Make sure to set the `ngrok-skip-browser-warning` request header with any value
to skip the interstitial warning page from ngrok: [More info](https://stackoverflow.com/questions/73017353/how-to-bypass-ngrok-browser-warning)
- Haven't tested out if the above bullet point is needed, try leaving it out to see if your GET requests work.

## Testing the server
- Use software like [Postman](https://www.postman.com/) to test the API's endpoints.

## Response schema
- All responses from all current and future API endpoints will be of the below JSON schema
- The examples below are of `POST /api/auth/sign-up`. See the `.go` files in the `handlers` directory for more info.
- The first example shows when the request succeeded and the second example otherwise.
- The `data` field in the response JSON when the request succeeded can be of any arbitrary schema, depending on which
endpoint you sent a request to. In the first example, the only field in the `data` field on the `POST /api/auth/sign-up`
endpoint is `token`
```json
{
    "success": true,
    "data": {
        "token": "..."
    }
}
```
```json
{
    "success": false,
    "message": "invalid credentials"
}
```

## Auth
- Persist the `token` received from the backend server during sign in or sign up in the Keychain.
- Check out [Storing Keys in the Keychain](https://developer.apple.com/documentation/security/storing-keys-in-the-keychain) on the Apple Developer website.
- Check out `KeychainHelper.swift` on our [hackathon project](https://github.com/Daetrail/BathHack2026/blob/main/BathHack2026/Helpers/KeychainHelper.swift).
- Tokens expire in three days after being generated by the backend server.
- For each request to a protected API endpoint (an endpoint that needs auth), set the `Authorization` header in your request with
`Bearer <token-from-server>`. Check out `NetworkService.swift` on our [hackathon project](https://github.com/Daetrail/BathHack2026/blob/main/BathHack2026/Services/NetworkService.swift).
